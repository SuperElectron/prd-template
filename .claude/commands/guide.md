Welcome to the PRD Template repo. Here's everything you need to know.

## First Time Here?

If you haven't set up yet, say **"help me get started"** and I'll walk you through it step by step (API keys, MCP servers, etc.).

## What You Can Do

| Command | What It Does |
|---|---|
| `/new-prd` | Create a new PRD from the template |
| `/new-decision` | Create a new decision record |
| `/new-research` | Create a research brief (with web research) |
| `/review-prd` | Review a PRD for completeness and quality |
| `/jtbd-analysis` | Analyze a PRD using the Jobs-to-be-Done framework |

**Tip:** You can pass context directly — e.g., `/new-prd notification preferences` — and I'll start drafting immediately.

## How This Repo Works

1. **Write** — Use a command above or ask me to help you draft a document
2. **Review** — Open a pull request on GitHub. Use `/review-prd` to self-check first
3. **Merge** — Once approved, merge to main. That PRD is now the accepted plan

Documents go in three folders:
- `prds/` — What to build and why
- `decisions/` — Key choices and trade-offs
- `research/` — Supporting data and analysis
