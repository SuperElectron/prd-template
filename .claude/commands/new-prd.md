Create a new PRD.

1. Read `docs/templates/prd-template.md` — use this exact structure, every section, in order.
2. Ask me: "What problem are we solving, and who is affected?"
3. Once I describe the problem, draft the full PRD following the template.
4. Name the file `prds/YYYY-MM-DD_short-description.md` using today's date.
5. Set Status to "Draft".

If I provide a topic upfront (e.g., `/new-prd notification preferences`), skip the question and draft immediately based on what I gave you. Ask clarifying questions only if the problem is genuinely unclear.
